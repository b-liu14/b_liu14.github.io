JavaScript基础练习题

  1. 返回值为undefined。因为匿名函数调用func.getNum时，作用域为匿名函数的参数Arguments所在作用域，this指针指向的是Arguments，因此返回this.num时返回的是undefined，那返回值的类型自然也是undefined。如果需要正常的返回number，可以在匿名函数中调用时使用如下三种方法：

    1. 使用call
    (function() {
    return arguments[0].call(func);
})(func.getNum)
    1. 使用apply
    (function() {
    return arguments[0].call(func);
})(func.getNum)
    1. 使用bind
    (function() {
    var f = arguments[0].bind(func);
    return typeof f();
})(func.getNum)

  2. console.log出的是undefined。因为foo函数中返回值为foo，即一个函数，而不是一个对象，因此在调用bar.x时由于函数中没有这个成员，当然返回的是undefined，若需要返回正常的1，则可以做如下修改：将返回值改为this，并将bar的初始化部分变为new foo。

var x = 0;
function foo(){
    debugger;
    x ++;
    this.x = x;
    return this;
}
var bar = new foo;
console.log(bar.x);
  1. alert的结果为function。因为JavaScript会在预编译阶段将var变量声明和函数声明提到作用域的最前面，（先提升var再提升函数声明），因此源代码等价于下面的代码。
function bar(){
    var foo;
    function foo(){}
    return foo;
    foo = 10;
    foo = '11';
}
alert(typeof bar());
  1. 依次alert出3,1。因为this具有晚绑定特性，当函数被直接调用时，this指向全局对象，因此第一个alert出的结果为全局中的x=3，第二个是正常的x=1。若第一个也需要返回正常的1，则可以使用call来实现（apply和bind亦可）。
var x = 3;
var foo = {
    x:2,
    baz: {
        x : 1,
        bar:function(){
            return this.x;
        }
    }
}
var go = foo.baz.bar;
var goo = foo;
alert(go.call(foo.baz));
alert(goo.baz.bar());

JavaScript进阶练习题

  1. 代码见src中的forecast.js。首先声明，本题思路是有我和室友宋知朋一起讨论出来的，但是我们讨论出思路之后便分头实现，最终他采用了递归的方式，我采用了递推的方式。

    本题的难点在于对于每一支队伍，其余的每一支队伍均有可能成为其对手，不能直接根据实力大小判定出该队伍后期的对阵情况，一切都只能用概率来描述。

    首先，我们定义每个一个元组$tuple$来表示小组中每个队伍成为小组冠军的概率，设元组有$l$（其中$l = 2^k, k \in N$）个元素，即该小组有$l$个队伍，每个元素都用一个对象$team$来表示，该对象具有$index$和$p$两个属性，分别代表该队伍在team_names中的下标和该队伍成为这$l$个队伍中的冠军的概率。此外，我们再定义一个二维数组$p[16][16]$， 其中$p[i][j]$代表队伍i战胜队伍j的概率。

    有了$tuple$和$p$的定义，接下来只需考虑初始化和合并元组两件事了。

    • 初始化时只需将16只队伍根据对阵图两两分为一组，计算其战胜对方的概率即可。由此，我们得到了8个元组，统一将其放入一个数组$tuples$中。

    • 接下来我们考虑两个元组的合并，若我们已经有了两个元组$tuple1$, $tuple2$，分别存储着两只队伍中每只队伍的$index$和其成为小组冠军的概率。现需要将这两个元组合并为一个元组$tuple$，储存这两个元组中每支队伍的$index$和其成为合并后的小组$tuple$的冠军的概率。显然，对于$tuple1$中的队伍$t1$，其成为$tuple$中的队伍中的冠军的概率可用如下公式表示： $$p = t1.p * (\sum_{t2 \in tuple2} t2.p * p[t1.index][t2.index])$$对于$tuple2$中的队伍也同理，因此我们只需要计算$tuple1, tuple2$中所有队伍成为$tuple$中的冠军并将其放入元组中，这样就完成了两个元组合并为一个元组的过程。

    • 接下来考虑将初始的8个元组合并为统一的一个元组的过程。这其实很简单，首先8个元组可以两两合并为一个长度为4的元组，这样便得到了4个长度为4的元组，然后继续两两合并，直到$tuples$中元组个数为一，则$tuples[0]$中便储存着所有队伍成为16支队伍中的冠军的概率。

    当然，本题代码中并未明确写明参赛队伍是16，而是通过获取队伍名称的长度来获取，因此也可以拓展至参赛队伍不等于16的情况。
